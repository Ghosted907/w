ملاحظات هامة
  • إذا كانت التذكرة أو القسيمة مصدرة عن طريق (مدى) يجب أخذ رقم بطاقة مدى التي أصدرت منها التذكرة/القسيمة + الاسم + الهوية.
  • إذا كانت التذكرة/القسيمة مصدرة بطرق دفع أخرى (بطاقات ائتمانية/حساب شركة) تُضاف بيانات الشركة ورقم السجل بدل الهوية/الإقامة.
  • لا تكون التذكرة صادرة من وكالة سياحية.
  • لا تكون التذكرة حكومية.
  • التأكد من حالة التذكرة قبل رفع الطلب (وفي حال تم إصدار بطاقة صعود الطائرة يجب إشعار الضيف بإلغاء بطاقة الصعود قبل طلب الاسترجاع).
  • إذا كان الجوال غير موجود في الحجز لا يتم قبول الطلب نهائياً ويُحوّل للاسترجاع عن طريق الموقع أو المحادثة الفورية.
  • إذا يوجد أكثر من مسافر ويريد استرجاع تذكرة واحدة: ترفق بيانات الضيف ورقم تذكرته فقط في الجدول.
  • إرفاق كامل بيانات الضيف (رقم الحجز/رقم التذكرة/رقم الجوال/IBAN/الهوية/الاسم...) والتنبيه أن أي تأخير برسوم يتحملها الموظف الذي أخر إرسال الطلب.
  • يتم حذف خطوط السير بعد رفع الطلب وبعد التأكد مع الضيف.

التغيب عن السفر (No-Show)
تذكرة بخط سير واحد
O: اختيار الخطوط وتسعير (يضاف رسوم تلقائياً)
S: اختيار الخطوط ثم حفظ + ريمارك + رفع طلب إصدار يدوي
Z: إبلاغ الضيف بفقدان التذكرة
U: اختيار الخطوط ثم حفظ + ريمارك + رفع طلب إصدار يدوي
تذكرة بخطين (ذهاب/عودة)
O: اختيار الخطوط وتسعير (يضاف رسوم تلقائياً)
S: اختيار الخطوط ثم حفظ + ريمارك + رفع طلب إصدار يدوي
Z: إبلاغ بفقدان قيمة الكوبون الأول ثم اختيار رحلات مناسبة + ريمارك + إصدار يدوي
U: اختيار الخطوط ثم حفظ + ريمارك + رفع طلب إصدار يدوي
التعديلات الإجبارية (UN / TK)
الشروط تعتمد على تعديل الإقلاع فقط. استخدم RHA أو RHI للمراجعة.
  • TK (تقديم الرحلة): يحق التعديل المجاني أو الاسترجاع الكامل بدون رسوم.
  • TK (تأخير أكثر من ساعتين): إذا نفس الدرجة استخدم TTP/ETRV/IVI/E#/S#/L#/RT، وإذا درجة مختلفة يتم رفع إيميل.
  • TK (تأخير 4 ساعات فأكثر): بديل على السعودية بنفس الدرجة أو استرجاع بدون رسوم.
  • UN (إلغاء تشغيلي) قبل 60 يوماً: بديل +/-7 أيام أو استرجاع بدون رسوم.
  • UN (إلغاء تشغيلي) خلال 60 يوماً: بديل على السعودية بنفس الدرجة أو استرجاع بدون رسوم.
الفصل (SPLIT)
لا يتم فصل الضيف المطلوب استرجاعه؛ يتم وضع المسافرين الآخرين فقط. لا يمكن فصل حجوزات السياحة أو الجروب.

أخطاء النظام (ERROR)
  • FOR RJT : OLD FOR REQUIRED → تأكد إدخال طريقة الدفع القديمة بشكل صحيح.
  • FOP RJT: NEED FORM OF PAYMENT → يجب إدخال طريقة الدفع الجديدة.
  • PREVIOUS TST EXISTS/SEGMENT OVERLAP → TTE/ALL ثم TMX/ALL.
  • DUPLICATE SEGMENT → احذف الرحلة القديمة بعد إبلاغ الضيف.
  • NEED RECEIVED FROM → احفظ التغييرات.
  • NUMBER OF SEGS/ETKT CPNS → تعديل رقم الكوبون.
  • SELECT FA OR FHE PNR LINE NUMBER → اختيار السطر الصحيح للتذكرة.
  • PROVIDE TICKET NUMBER TO EXCHANGE → أدخل رقم التذكرة ثم FXQ/S#.
  • NEED TICKETING ARRANGMENT → TKOK.
  • ITINERARY DISCREPANCY BETWEEN TKTS-TRY SEPARATELY → FXQ/S#/T#.
أخطاء ربط التذكرة (FHE)
  • OVERRIDE RESTRICTED → احذف O من إدخال الربط.
  • NO MATCH FOR BOARD/OFF POINTS → أدخل رحلة وهمية مطابقة ثم احذفها بعد الربط.
  • NO MATCH FOR BOOKING CLASS → أدخل رحلة وهمية بنفس الدرجة ثم احذفها.
  • PAX IN PNR DOES NOT MATCH → عدّل الاسم بالحجز ليطابق التذكرة.
  • NUMBER OF SEGS/ETKT CPNS DO NOT MATCH → طابق أول كوبون مع أول سطر رحلة.

قاموس الرموز (Placeholders)

  • P# = رقم المسافر (Passenger Number) مثل P1
  • S# = رقم الرحلة/الرحلة داخل الحجز (Segment Number) مثل S2
  • L # = رقم السطر (Line Number) غالبًا من RTF أو قائمة الخدمات
  • TKT# = رقم التذكرة/السطر الخاص بها حسب السياق
  • EMD# = رقم القسيمة/EMD
  • ### = أرقام ناقصة حسب المطلوب
  • FF = كود الباقة (Package) مثل NSAVERE / NBASICE / NSEMIFLEXE / NFLEXE
  • FPO = وسيلة دفع قديمة/سابقه (Old Form of Payment)
  • CC = بطاقة بنكية (Credit Card)
